# Критерии AI-скрининга заявок моделей

> Источник: docs/model-application-guide.md

---

## Обязательные поля заявки

| # | Поле | Что указывать | Критичность |
|---|---|---|---|
| 1 | **Имя** | Только имя (фамилия НЕ обязательно) | Высокая |
| 2 | **Возраст / дата рождения** | Точная дата не нужна. Достаточно знать возраст — вписать год (день/месяц случайные) | Высокая |
| 3 | **Telegram** | Юзернейм, если есть. Преимущество — интервьюер связывается через TG | Средняя |
| 4 | **Фото** | В полный рост, стоя. Если по лицу видно, что худая — достаточно фото лица | Высокая |
| 5 | **Модель устройства** | Точная модель! НЕ просто "iPhone" или "Android" | **Критическая** |
| 6 | **Мессенджер кандидата** | TG-юзер или номер WA (формат ссылки `wa.me/...`) | Высокая |
| 7 | **Краткое описание** | Что рассказал о работе, обязанности, условия — вкратце | Средняя |

## Дополнительные поля (влияют на одобрение)

| Поле | Что указывать | Влияние |
|---|---|---|
| **Дополнительное устройство** | Есть ли ноутбук/ПК + веб-камера | Если есть — запись 100% одобрят |
| **Параметры фигуры** | "Худая" или рост/вес | Формула: рост - вес >= 100 → одобрят |
| **Как представились** | Если была легенда (от лица действующей модели/продюсера) | Обязательно указать, чтобы интервьюер не попал впросак |
| **Вакансия** | "Модель со смартфона" / "Модель" | Некритично |
| **Языки** | Рабочие языки кандидата | Влияет на выбор платформы |
| **Приложение для собеса** | Google Meet / Zoom / TG | Удобство кандидата |

## Критерии внешности

### Формула веса

```
рост (см) - вес (кг) >= 100 → ОДОБРЕНИЕ
рост (см) - вес (кг) < 100 → вероятный ОТКАЗ или HOLD
```

**Примеры:**
- 170 см, 65 кг → 170 - 65 = 105 → Одобрение
- 165 см, 60 кг → 165 - 60 = 105 → Одобрение
- 160 см, 63 кг → 160 - 63 = 97 → Отказ / Hold
- 175 см, 70 кг → 175 - 70 = 105 → Одобрение

### Дополнительные факторы

| Фактор | Решение |
|---|---|
| **Полнота** | Главный фактор отказа |
| **Спорные фото** | Согласовывают с партнёром, но могут отказать сразу |
| **Чернокожие девушки** | Часто отказ, если не "прям красивая" |
| **Пограничные случаи** | Запросить 2-3 фото в полный рост для дополнительной оценки |

### Требования к фото

- В полный рост, стоя — обязательно
- Без фильтров и ретуши
- Актуальные (не старше 3 месяцев)
- Если по лицу видно, что худая — допустимо только фото лица

Подробный гайд: `knowledge/models/photo-guide.md`

## Логика одобрения устройств

### Смартфон

| Устройство | Результат |
|---|---|
| iPhone 11+ | Одобрение |
| iPhone X, XR, XS, 8 и старше | Отказ |
| "iPhone" без модели | **Отказ** (уточнить) |
| Android 2023+ | Одобрение |
| Android до 2023 | Отказ |
| "Android" без модели/года | **Отказ** (уточнить) |

### Альтернатива

| Устройство | Результат |
|---|---|
| Ноутбук/ПК + веб-камера | **100% одобрение** |
| Смартфон + ноутбук с камерой | Максимальный приоритет |

### Отказы по устройству

| Ситуация | Результат |
|---|---|
| Только "iPhone" без указания модели | Отказ |
| Обещание купить устройство после собеса | Отказ |
| Старый телефон + нет ноутбука | Отказ |
| Низкая скорость интернета, но готов улучшить | **ОК** (единственное исключение) |

## Возраст

| Условие | Результат |
|---|---|
| 18-25 лет | Стандартное одобрение |
| < 18 лет | Безусловный отказ |
| > 25 лет | На усмотрение (зависит от внешности и рынка) |

## Статус Hold

**Hold** — удержание: модель не уволена, но смены не ставят. Причины:

| Причина Hold | Описание |
|---|---|
| Нехватка операторов | Нет свободного оператора для модели |
| Спорная внешность | Партнёр не уверен — ждут решения |
| Нет доп. устройства | Есть только телефон, хотели бы ноутбук + камера |

Hold — это не отказ. Модель может быть активирована позже.

## Что проверять в заявке (чеклист)

### Обязательная проверка

- [ ] Указана конкретная модель устройства (не просто "iPhone" / "Android")
- [ ] Устройство соответствует минимальным требованиям
- [ ] Возраст 18+
- [ ] Есть фото (в полный рост или по лицу видно, что худая)
- [ ] Указан контакт (TG или WA)
- [ ] Указано имя

### Дополнительная проверка (повышает шанс одобрения)

- [ ] Есть ноутбук/ПК с веб-камерой (= 100% одобрение по устройству)
- [ ] Параметры фигуры: рост - вес >= 100
- [ ] Указана легенда (как представились кандидату)
- [ ] Указан предпочтительный мессенджер
- [ ] Указаны рабочие языки

## Результаты скрининга

| Результат | Когда |
|---|---|
| **APPROVE** | Устройство ОК + внешность ОК + возраст 18+ |
| **REJECT** | Устройство не подходит / возраст < 18 / явный несоответствие формату |
| **HOLD** | Пограничный случай — нужно дополнительное фото / устройство / решение партнёра |
| **REQUEST INFO** | Не хватает данных — уточнить модель устройства, запросить фото и т.д. |

**Важно:** Approve/Reject — ТОЛЬКО ручной (кнопка в admin-чате). AI не авто-аппрувит. AI только предлагает рекомендацию, финальное решение за админом.
